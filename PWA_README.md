# EvrikaEdu PWA - Progressive Web Application

## Описание

EvrikaEdu теперь является полноценным PWA приложением, оптимизированным для мобильных устройств с поддержкой:
- ✅ Установка на главный экран (iOS и Android)
- ✅ Работа в offline режиме
- ✅ Кэширование данных и ресурсов
- ✅ Push-уведомления (готово к расширению)
- ✅ Адаптивный UI для мобильных устройств
- ✅ Touch-friendly интерфейс

## Основные возможности PWA

### 1. Установка приложения

**Android:**
- Откройте сайт в Chrome
- Нажмите на меню (три точки)
- Выберите "Установить приложение" или "Добавить на главный экран"
- Или используйте всплывающее уведомление внизу экрана

**iOS (Safari):**
- Откройте сайт в Safari
- Нажмите кнопку "Поделиться" (квадрат со стрелкой)
- Выберите "На экран Домой"
- Нажмите "Добавить"

### 2. Offline режим

Приложение автоматически кэширует:
- Все статические ресурсы (JS, CSS, изображения)
- API запросы для последующего использования
- Страницы для навигации без интернета

**Стратегии кэширования:**
- **Network First** - для основных страниц (сначала сеть, потом кэш)
- **Cache First** - для статических ресурсов (сначала кэш, потом сеть)
- **API Network First** - для API с timeout 5 секунд

### 3. Индикаторы состояния

- **Offline индикатор** - показывает когда нет подключения к интернету
- **Уведомление о восстановлении** - сообщает о восстановлении соединения
- **Промпт установки** - предлагает установить приложение на устройство

## Технические детали

### Service Worker

Файл: `src/service-worker.js`

**Функции:**
- Кэширование ресурсов при установке
- Обновление кэша при активации
- Обработка fetch запросов с fallback
- Поддержка background sync
- Готовность к push-уведомлениям

### Манифест

Конфигурация в `vite.config.js`:
- Имя: EvrikaEdu Platform
- Короткое имя: EvrikaEdu
- Режим отображения: standalone
- Ориентация: portrait
- Тема: #6366f1 (индиго)

### Иконки

Размеры:
- 192x192 - для Android
- 512x512 - для высоких разрешений
- Поддержка maskable для адаптивных иконок

## Мобильная оптимизация

### Touch-friendly элементы

- Минимальный размер кнопок: 44x44px (Apple HIG)
- Увеличенные отступы для touch targets
- Предотвращение случайных нажатий
- Отключение highlight при нажатии

### Адаптивный дизайн

**Breakpoints:**
- Mobile: < 768px
- Tablet: 768px - 1024px
- Desktop: > 1024px

**Оптимизации:**
- Уменьшенные отступы на мобильных
- Адаптивная навигация (hamburger menu)
- Оптимизированные размеры шрифтов
- Безопасные зоны для iOS (notch)

### Производительность

- Lazy loading изображений
- Code splitting
- Минификация ресурсов
- Gzip/Brotli сжатие
- Prefetching критических ресурсов

## Разработка

### Локальное тестирование PWA

1. Запустите dev сервер:
```bash
cd frontend
npm run dev
```

2. Откройте Chrome DevTools:
   - Application → Service Workers
   - Application → Manifest
   - Lighthouse → PWA audit

3. Тестирование offline:
   - DevTools → Network → Offline checkbox
   - Или Service Workers → Offline checkbox

### Production build

```bash
cd frontend
npm run build
npm run preview
```

### Проверка PWA

**Chrome DevTools Lighthouse:**
- Progressive Web App score должен быть > 90
- Проверка всех критериев PWA
- Performance audit

**Тестирование на реальных устройствах:**
- Android: Chrome, Samsung Internet
- iOS: Safari (ограниченная поддержка PWA)

## Обновления приложения

Service Worker автоматически:
1. Проверяет обновления при каждом визите
2. Скачивает новую версию в фоне
3. Активирует после закрытия всех вкладок
4. Или обновляет при следующем визите

**Принудительное обновление:**
- Закройте все вкладки приложения
- Откройте снова - новая версия активируется

## Известные ограничения

### iOS Safari

- Ограниченная поддержка Service Workers
- Нет автоматического промпта установки
- Ограничения на размер кэша (50MB)
- Кэш может быть очищен системой

### Android

- Полная поддержка PWA
- Автоматические промпты установки
- Больший размер кэша
- Лучшая интеграция с системой

## Мониторинг

### Логи Service Worker

Откройте консоль браузера для просмотра:
- `[SW]` - логи service worker
- `[PWA]` - логи PWA функций

### Метрики

Отслеживайте:
- Процент установок
- Использование offline режима
- Ошибки service worker
- Время загрузки ресурсов

## Безопасность

- HTTPS обязателен для PWA
- Service Worker работает только на HTTPS
- Кэш изолирован по origin
- Защита от XSS через CSP

При проблемах с PWA:
1. Очистите кэш браузера
2. Удалите и переустановите приложение
3. Проверьте консоль на ошибки
4. Убедитесь что используете HTTPS
